# Инструкция по развертыванию (Deploy Guide)

## 🛠 Требования

- Python 3.11+
- PostgreSQL 14+
- Два сервера (или две VM): один в РФ (Trusted Zone), один за рубежом (DMZ/Shunt).

## 1. Настройка Российского сегмента (RF-Core)

### Установка зависимостей

```bash
cd rf-core
pip install -r requirements.txt
```

### Запуск FastAPI

```bash
uvicorn server_rf:app --host 0.0.0.0 --port 8000
```

**Рекомендация:** Настроить Whitelist IP (разрешить доступ только с IP казахстанского сервера).

### Создание базы данных

База данных создается автоматически при первом запуске FastAPI. Файл `database.db` появится в корне папки.

## 2. Настройка Зарубежного сегмента (KZ-Gateway)

### Установка зависимостей

```bash
cd kz-gateway
pip install -r requirements.txt
```

### Запуск бота

```bash
./venv/bin/python bot_local.py
```

## 🔗 Связь между сегментами (альтернативные сценарии)

В MVP возможны два сценария развёртывания:

### Сценарий А: Оба компонента на одном VPS (рекомендуется для MVP)

Самый простой вариант. FastAPI и бот работают на одном сервере. Бот стучится на `http://127.0.0.1:8000` локально.

```bash
# Терминал 1: FastAPI
./venv/bin/uvicorn server_rf:app --host 0.0.0.0 --port 8000

# Терминал 2: Бот
./venv/bin/python bot_local.py
```

### Сценарий Б: Reverse SSH Tunnel (для разработки)

Используется, когда FastAPI крутится на локальной машине разработчика (например, Windows), а бот — на VPS.

**На машине разработчика (Windows/Linux/Mac):**

```bash
ssh -i /path/to/your/key.pem -R 8000:127.0.0.1:8000 root@<IP_VPS>
```

Эта команда пробрасывает локальный порт 8000 на VPS через SSH-туннель.

**На VPS:**

```bash
# Бот стучится на http://127.0.0.1:8000, 
# трафик идёт через туннель на локальную машину разработчика
./venv/bin/python bot_local.py
```

**Важно:**
- SSH-окно должно быть открыто всё время работы
- Для продакшена используйте `autossh` или systemd-сервис для автопереподключения туннеля
- В продакшене туннель заменяется на **mTLS** (см. раздел "Безопасность канала")

### Сценарий В: Production (mTLS)

Для промышленной эксплуатации reverse SSH tunnel заменяется на **Mutual TLS** с выпуском сертификатов от собственного CA. Подробности см. в пояснительной записке к ВКР.

## 🔐 Безопасность канала связи (MVP vs Prod)

### MVP (текущая версия)
- SSH-туннель или прямой HTTPS
- Быстро для разработки
- Не для продакшена

### Production (рекомендуется)
- **mTLS** (взаимная аутентификация)
- Whitelist IP на firewall
- Сертификаты от собственного CA
- Полное соответствие принципам Zero Trust
