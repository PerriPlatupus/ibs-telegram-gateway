# Secure Telegram Bot — Архитектура защищенного шлюза

## 🎯 Что это?
Прототип программного комплекса для безопасной обработки персональных данных (ПДн) сотрудников через мессенджер Telegram. Система спроектирована с учетом требований **152-ФЗ «О персональных данных»** и принципов **Zero Trust Architecture**.

**Ключевая особенность:** Разделение на два контура доверия позволяет использовать иностранный API (Telegram) без нарушения требования о локализации БД на территории РФ.

## 🏗️ Архитектура
Система состоит из двух изолированных сегментов:
1. **Зарубежный шлюз (KZ):** Принимает трафик от Telegram API. Не хранит ПДн на диске (`MemoryStorage`). Работает по принципу Long Polling (нет открытых входящих портов).
2. **Доверенный сегмент (RF):** FastAPI + PostgreSQL. Здесь происходит валидация данных, хранение БД и криптографическая фиксация согласий.

## ✅ Соответствие 152-ФЗ
- **Локализация (ч. 5 ст. 18):** Первичная запись ПДн ведется только в БД на сервере в РФ.
- **Трансграничная передача (ч. 2 ст. 12):** Реализован механизм информированного согласия с указанием стран (БВО, ОАЭ).
- **Неотрекаемость (ч. 2 ст. 9):** Согласие фиксируется через кортеж `<TG ID, timestamp, hash PDF>`.
- **Минимизация рисков:** Валидация входных данных через Pydantic защищает от SQLi/XSS.

## ⚖️ Важное уведомление
Данный проект является **MVP (минимально жизнеспособным продуктом)**. Для промышленной эксплуатации оператору ПДн необходимо самостоятельно выполнить юридические процедуры (уведомление РКН, издание внутренних приказов). Чек-лист необходимых действий см. в [COMPLIANCE_CHECKLIST.md](COMPLIANCE_CHECKLIST.md).
